# Автодеплой бота из GitHub (Python, без переписывания)

Цель: правишь код → пуш в GitHub → хостинг **сам** пересобирает и перезапускает бота.
Никаких ручных действий на сервере. Бот остаётся на Python (aiogram), деплой — через `Dockerfile`.

Рекомендуемая площадка — **Timeweb Cloud** (РФ, оплата картами РФ). Альтернатива — **Amvera**.
Обе умеют автодеплой из GitHub по `Dockerfile`, который уже лежит в репозитории.

> Для **Yandex Cloud** (VM + GitHub Actions) — отдельная инструкция: `YANDEX_DEPLOY.md`.

---

## Что уже готово в репозитории
- `Dockerfile` — как собрать и запустить бота (универсальный, для любого Docker-хостинга).
- `.dockerignore` — чтобы в образ не попали лишние файлы и секреты.
- `requirements.txt` — зависимости.
- `welcome.jpg` — картинка приветствия (попадает в образ автоматически).

**Токен и прочие секреты в репозиторий НЕ кладём.** Они задаются в панели хостинга
как переменные окружения (см. ниже). Файл `.env` нужен только для локального запуска.

---

## Переменные окружения (задать в панели хостинга)
| Переменная | Значение | Обязательно |
|---|---|---|
| `BOT_TOKEN` | токен от @BotFather | **да** |
| `WELCOME_IMAGE` | `welcome.jpg` | да (иначе приветствие без картинки) |
| `CHANNEL` | `@sorokavmarketinge` | нет (есть в коде по умолчанию) |
| `CHANNEL_URL` | `https://t.me/sorokavmarketinge` | нет |

> Тексты сообщений уже зашиты в код по ТЗ — переопределять не нужно.

## Вариант A — Timeweb Cloud (рекомендую)
1. Зарегистрируйтесь на timeweb.cloud, пополните баланс.
2. **Apps / Приложения → Создать → из GitHub.** Привяжите GitHub-аккаунт,
   выберите репозиторий `dkaratsapov-web/soroka_bot` и ветку для деплоя.
3. Тип сборки — **Dockerfile** (определится автоматически, раз файл есть в корне).
4. В разделе **Переменные окружения** добавьте `BOT_TOKEN` и `WELCOME_IMAGE=welcome.jpg`.
5. Создайте приложение. Дальше **каждый пуш в выбранную ветку = автоматическая пересборка и перезапуск.**

## Вариант B — Amvera
1. Зарегистрируйтесь на amvera.ru, пополните баланс.
2. **Создать проект → подключить GitHub** (или их git-remote), выбрать репозиторий/ветку.
3. Тип окружения — **Docker** (использует наш `Dockerfile`).
4. В настройках проекта добавьте переменные `BOT_TOKEN` и `WELCOME_IMAGE=welcome.jpg`.
5. Запуск. Дальше пуш в ветку → автодеплой.

## Про ветку
Хостинг привязывается к одной ветке. Сейчас разработка идёт в ветке
`claude/affectionate-johnson-37hlcy`, основная ветка репозитория — `main`.
Варианты:
- Привязать хостинг прямо к рабочей ветке `claude/affectionate-johnson-37hlcy` — задеплоится сразу.
- Либо влить изменения в `main` (через Pull Request) и деплоить с `main` — «правильнее» для прода.

Скажите, какой путь предпочитаете — помогу оформить.

## Обязательный шаг до запуска
Сделайте бота **администратором** канала `@sorokavmarketinge` — без этого он не сможет
проверять подписку (`getChatMember` вернёт ошибку, и подписка не будет видна).

## Как выглядит «максимальная автоматизация правок»
После подключения автодеплоя цикл внесения изменений такой:

1. Вы говорите, что поменять (текст, кнопку, логику).
2. Я правлю код, коммичу и пушу в GitHub.
3. Хостинг сам пересобирает образ и перезапускает бота — через ~1–2 минуты правка «в проде».

То есть ручных действий с вашей стороны после первичной настройки — ноль.
