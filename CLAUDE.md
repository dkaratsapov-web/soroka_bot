# Проект: бот «лид-магнит за подписку» (Telegram + позже MAX)

Это контекст для Claude Code. Прочитай его целиком перед работой.

## Цель проекта
Бот выдаёт бесплатный материал (лид-магнит: файл / ссылку на вебинар) в обмен на
подписку на канал. Нужен в Telegram, а вторым этапом — в мессенджере MAX (МАКС).
Делаем без платных конструкторов: бот простой и типовой.

## Текущий статус
- ✅ **Telegram-бот готов** — `leadmagnet_bot.py` (фреймворк aiogram 3.x).
  Логика: `/start` → проверка подписки через `getChatMember` → выдача материала.
- 🟡 Деплой (запуск 24/7) — артефакты готовы (Docker + systemd, см. `DEPLOY.md`),
  осталось запустить на хосте с доступом к Telegram.
- ⬜ MAX-версия — не начата (есть важные ограничения, см. ниже).

## Файлы
- `leadmagnet_bot.py` — основной бот, конфиг читается из `.env`.
- `.env.example` — шаблон переменных окружения. Реальный `.env` создаётся при настройке.
- `requirements.txt` — зависимости (`aiogram`, `python-dotenv`).
- `setup.sh` — первичная настройка (venv + зависимости + создание `.env`).
- `Dockerfile`, `docker-compose.yml`, `.dockerignore` — деплой через Docker.
- `deploy/leadmagnet-bot.service` — systemd-юнит для VPS (автоперезапуск).
- `DEPLOY.md` — инструкция по запуску 24/7 (Docker и systemd).
- `CLAUDE.md` — этот файл.

## Как запустить локально
1. `bash setup.sh` (создаст venv, поставит зависимости, сделает `.env` из шаблона).
2. Заполнить `.env`: `BOT_TOKEN` (от @BotFather), `CHANNEL`, `CHANNEL_URL`, текст/файл материала.
3. Сделать бота **администратором** канала — иначе проверка подписки не работает.
4. `source venv/bin/activate && python leadmagnet_bot.py`

## Важные ограничения по MAX (проверить перед обещаниями заказчику)
- Опубликовать бота в MAX можно только от **верифицированного юрлица/ИП — резидента РФ**.
  Физлицам/самозанятым доступа может не быть. Модерация бота — до ~48 часов.
- API MAX беднее телеграмовского. **Не факт, что есть аналог `getChatMember`** (проверки
  подписки на канал). Перед реализацией свериться с актуальной докой на https://dev.max.ru
  и, если проверки подписки нет, согласовать упрощённую механику
  («нажал кнопку → получил материал»).
- Прод в MAX — только через Webhook (long polling для прод не подходит).

## Конвенции и важные правила
- **Не коммить `.env` и токены** в git. Добавить `.gitignore` (venv/, .env, __pycache__/).
- Конфиг — только через переменные окружения, не хардкодить секреты.
- Код держать простым и читаемым: проект должен оставаться «типовым» и поддерживаемым.
- Сообщения пользователю — на русском.
- Любой деструктивный шаг (деплой, изменение токена, отправка реальных сообщений
  пользователям) — сначала согласовать с человеком.
